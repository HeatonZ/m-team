# Task intake mode

Use this mode immediately after claim.

## Purpose

Build a correct local picture of the current baton before you do any work.

## Read first

Use the current task view to confirm at least:

- `taskType`
- current `description`
- current `stepContract`
- recent context
- any already-produced files or data you can reuse

## Intake checks

Confirm these points before acting:

1. The exact action required by the current `description`.
2. The result implied by `expectedOutcome`.
3. The checks in `doneWhen` that define baton completion.
4. The files, data, or intermediate results from recent context that can be reused.
5. The work that is outside the scope of this baton.

## Hard rules

- Do not restate the whole task goal as your action plan.
- Do not redo earlier completed batons unless the current baton explicitly says to repair or replace them.
- Do not invent hidden requirements that are not in the current baton or contract.
- Do not start with a broad plan for the whole task chain.

## Good intake result

At the end of intake, you should be able to state in one sentence:

- what you need to do now,
- what result counts as done now,
- and what prior artifact you will build on.
