# M7：并发与占用

---

## M7-1：两个 agent 并发认领同一任务，只能成功一个

**场景描述：** 两个 agent 同时尝试认领同一个 `pending + ready` 任务，系统必须保持原子性，只允许一个成功进入执行态。

**测试步骤：**

1. Publisher 发布一个待认领任务
2. alice 和 bob 几乎同时调用认领动作
3. 验证只有一个认领成功
4. 验证另一个收到失败结果，原因是任务已不再处于可认领状态
5. 查询任务，确认执行人是成功认领的那个 agent

---

## M7-2：已有活跃任务的 agent 不能再接新任务

**场景描述：** 一个 agent 已持有运行中任务时，不应再被分配第二个任务。

**测试步骤：**

1. 发布两个任务 t1、t2
2. alice 认领 t1，进入 `running + executing`
3. 查询 `getPendingTasks('alice')`
4. 验证返回空列表
5. 查询 `getPendingTasks('bob')`
6. 验证 bob 仍能看到待认领任务
7. 让 alice 直接尝试认领 t2
8. 验证认领失败

---

## M7-3：风险任务与普通任务并存时，不影响可认领原子性

**场景描述：** 看板上同时存在普通待接任务和风险任务时，并发认领逻辑仍然只按可认领状态原子处理，不因 risk 展示层而破坏 claim 行为。

**测试步骤：**

1. 构造一个普通 `ready` 任务和一个 `reworking` 风险任务
2. 两个 agent 同时争抢同一个 `ready` 任务
3. 验证 claim 结果仍然只有一个成功
4. 验证另一个任务的风险状态不受影响
