# TC-G：并发场景

---

## TC-G1：两个 Agent 同时认领同一任务

**场景描述：** 两个 Agent 并发调用 claimTask 认领同一个 PENDING 任务，race condition 下只有一个能成功。

**测试步骤：**

1. Publisher 发布一个待认领任务
2. alice 和 bob 同时（并发）调用认领接口，认领同一个任务
3. 验证其中一个认领成功，另一个认领失败
4. 失败的那个原因应为"任务不是待认领状态"（已被第一个认领）
5. 查询任务，确认执行人为成功认领的那个 Agent

---

## TC-G2：Agent 已有活跃任务不再分配新任务

**场景描述：** Agent 认领一个任务还未完成时，不应再被分配新任务，防止一个 Agent 同时跑多个任务。

**测试步骤：**

1. Publisher 发布两个任务 t1 和 t2
2. alice 认领 t1，t1 状态变为执行中
3. 查询 alice 的待认领任务，返回空列表（因为 alice 已有活跃任务）
4. 查询 bob 的待认领任务，返回至少 1 条（bob 无活跃任务，可正常分配）
5. alice 尝试再次认领 t2
6. 验证认领失败（因为 alice 已有活跃任务）
