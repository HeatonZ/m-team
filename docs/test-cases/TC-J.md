# TC-J：优先级调度

---

## TC-J1：高优先级任务优先返回

**场景描述：** getPendingTasks 应按优先级排序返回，高优先级任务排在前面。

**测试步骤：**

1. 设置工作空间根目录
2. 按顺序发布三个任务：普通任务（normal）、紧急任务（high）、低优任务（low）
3. 调用待认领任务查询
4. 验证返回顺序为：紧急任务在最前、普通任务在中间、低优任务在最后（high > normal > low）

---

## TC-J2：同一优先级按创建时间先来先服务

**场景描述：** 同优先级的任务，按创建时间升序返回（先创建的先被分配）。

**测试步骤：**

1. 先发布任务"先发"，稍后发布任务"后发"
2. 查询待认领任务
3. 验证"先发"的任务排在前面，"后发"的任务排在后面
