# TC-C：任务失败流程

---

## TC-C1：Agent 执行失败并标记失败

**场景描述：** Agent 认领后遇到不可恢复错误（如网络不可达），调用失败接口标记任务。

**测试步骤：**

1. Publisher 发布任务，alice 认领成功
2. alice 追加上下文记录尝试连接外部服务的步骤，输出中包含错误信息"网络不可达"
3. alice 调用失败接口，传入错误信息"网络不可达"和失败步骤上下文
4. 验证失败成功，任务状态变为失败，完成时间被记录，执行人被清空，上下文长度为 3
5. 查询任务，确认状态为失败、完成时间不为空

---

## TC-C2：失败后再次调用失败

**场景描述：** Agent 调用失败后，残留的旧调用再次触发失败接口，系统应拒绝。

**测试步骤：**

1. Publisher 发布任务，alice 认领
2. alice 调用失败接口，任务变为失败状态
3. 再次调用失败接口
4. 验证返回失败，原因是"任务不在执行中"，任务状态保持为失败不变

---

## TC-C3：认领后未开始直接失败

**场景描述：** Agent 认领后还没做任何操作就发现无法执行，直接标记失败。

**测试步骤：**

1. Publisher 发布任务，alice 认领成功
2. alice 调用失败接口，不传任何上下文步骤
3. 验证失败成功，任务状态变为失败
4. 验证上下文长度为 1（无中间步骤记录）
