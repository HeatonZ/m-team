# TC-E：放弃任务流程

> 与 Relay 的区别：relinquish 不追加 context 步骤，任务直接放回 PENDING，不记录交接动作。

---

## TC-E1：Agent 放弃后另一个 Agent 完成

**场景描述：** Agent alice 认领后主动放弃，bob 接手完成。relinquish 不追加上下文（与 relay 的区别）。

**测试步骤：**

1. Publisher 发布任务，alice 认领
2. alice 追加上下文步骤"做了部分工作"
3. 验证上下文长度为 2
4. alice 调用放弃接口（不传上下文步骤）
5. 验证放弃成功，任务状态变为待认领，执行人被清空，上一步执行人记录为 alice
6. 查询任务，确认上下文长度仍为 2（放弃不追加新上下文，与 relay 的关键区别）
7. bob 认领该任务，认领成功
8. bob 完成
9. 验证完成成功，任务变为已完成，上下文长度为 3（只有 bob 完成时追加的那一条）

---

## TC-E2：非当前 Executor 放弃失败

**场景描述：** alice 认领的任务，bob 尝试放弃，应被拒绝。

**测试步骤：**

1. Publisher 发布任务，alice 认领成功
2. bob 调用放弃接口
3. 验证放弃失败，原因为"不是当前执行人"
4. 查询任务，确认执行人仍为 alice，状态仍为执行中，任务未被改变

---

## TC-E3：CANCELLED 任务不可放弃

**场景描述：** 任务被取消后，executor 尝试放弃应被拒绝。

**测试步骤：**

1. Publisher 发布任务，alice 认领
2. Publisher 取消任务，任务变为已取消
3. alice 调用放弃接口
4. 验证放弃失败，原因为"任务已取消"
5. 查询任务，确认状态保持为已取消
